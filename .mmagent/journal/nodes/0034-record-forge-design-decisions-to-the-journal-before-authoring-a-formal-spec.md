## Context

PROCESS NOTE (Forge, user preference 2026-06-08): the user prefers recording
design decisions to the **MMA journal** ("think before you do") **INSTEAD** of
writing a `docs/superpowers` spec file at this stage.

Capture conclusions in the journal first; **defer formal spec/plan authoring**.

## Consequences

- For the current Forge stage, write decisions here (journal nodes), not as a
  superpowers spec doc.
- Formal spec/plan authoring is deferred, not skipped — the build plan (0033)
  still calls for per-sub-project specs once we move into implementation.
- When in doubt about whether to record vs. spec a Forge conclusion, default to
  the journal at this phase.

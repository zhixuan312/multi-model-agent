# Migration — 2.x MCP to 3.x HTTP REST

`@zhixuan92/multi-model-agent-mcp` was renamed to
`@zhixuan92/multi-model-agent` in 3.0.0. All MCP transports have been
removed; the package is now a standalone HTTP service with
client-installable skills.

## Minimal migration steps

```bash
npm uninstall -g @zhixuan92/multi-model-agent-mcp
npm install -g @zhixuan92/multi-model-agent
mmagent serve                                             # start the daemon
mmagent install-skill --target=claude-code                # install skills (all by default)
```

After a later `npm install @zhixuan92/multi-model-agent` upgrade, the
`postinstall` hook automatically refreshes any previously installed skills
via `mmagent update-skills --if-exists`.

## Endpoint mapping

| Old MCP tool               | New REST endpoint                |
|---                         |---                               |
| `delegate_tasks`           | `POST /delegate?cwd=<abs>`       |
| `audit_document`           | `POST /audit?cwd=<abs>`          |
| `review_code`              | `POST /review?cwd=<abs>`         |
| `verify_work`              | `POST /verify?cwd=<abs>`         |
| `debug_task`               | `POST /debug?cwd=<abs>`          |
| `execute_plan`             | `POST /execute-plan?cwd=<abs>`   |
| `retry_tasks`              | `POST /retry?cwd=<abs>`          |
| `register_context_block`   | `POST /context-blocks?cwd=<abs>` |
| `get_batch_slice`          | `GET /batch/:id?taskIndex=N`     |
| `confirm_clarifications`   | `POST /clarifications/confirm`   |

## Polling shape (3.1.0)

`GET /batch/:id` returns:

- `202 text/plain` while the batch is pending — the body is the current
  running headline (e.g. `1/1 running, 47s elapsed, worker: MiniMax-M2.7`).
- `200 application/json` once the batch is terminal — the body is the
  uniform 7-field envelope:

  ```json
  {
    "headline": "...",
    "results": [...] | {"kind": "not_applicable", "reason": "..."},
    "batchTimings": {...} | NotApplicable,
    "costSummary": {...} | NotApplicable,
    "structuredReport": {...} | NotApplicable,
    "error": {"code": "...", "message": "..."} | NotApplicable,
    "proposedInterpretation": "..." | NotApplicable
  }
  ```

See `packages/server/src/skills/_shared/polling.md` for the reference POSIX
polling loop.

## Identity probe

`mmagent info` prints the CLI version, bind/port, token fingerprint, and —
if the daemon is reachable — its version / pid / uptime. Scripts should use
`mmagent info --json` to discover `{port, tokenFingerprint, running}`.

See [CHANGELOG 3.0.0](https://github.com/zhixuan312/multi-model-agent/blob/master/CHANGELOG.md)
for the full breakage list and 3.1.0 for the polling shape change rationale.
